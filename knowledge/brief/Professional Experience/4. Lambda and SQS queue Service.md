### Key Idea

I use AWS Lambda with SQS (Simple Queue Service) to handle asynchronous and background processing in a scalable way.

### What Matters

- application receives request
- pushes message to SQS queue
- SQS triggers Lambda
- Lambda processes the message
- result is stored or next action is triggered

### Quick Summary

Instead of doing heavy work inside a request: 👉 I push tasks to SQS queue, and let Lambda process them in the background
