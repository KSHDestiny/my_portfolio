### Key Idea

I design systems to maintain data consistency when a single user action triggers multiple dependent backend steps.

### What Matters

- parsing input (e.g., files)
- creating records
- updating workflow states
- writing related data
- triggering downstream processes

### Quick Summary

In real-world features, one request often performs multiple operations, such as: If one step fails while others succeed, the system can end up in a partially inconsistent state, which is difficult to recover from.
