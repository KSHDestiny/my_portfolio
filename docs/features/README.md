# Features README

## Purpose
Track active and planned work items so agents can quickly load only relevant context.

## How to Use
1. Create one note per active workstream in `docs/features/`.
2. Keep status updated (`planned`, `in-progress`, `blocked`, `done`).
3. When done, add a summary entry to `ARCHIVE.md`.

## Active Features
- None yet.

## Suggested Note Naming
- `feature-<short-name>.md`
- `bugfix-<short-name>.md`
- `refactor-<short-name>.md`

## Required Fields for Each Feature Note
- Title
- Status
- Owner
- Goal
- Scope (in/out)
- Files touched
- Validation steps/results
- Risks/follow-ups
